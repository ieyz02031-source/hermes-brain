# Contributing

Thank you for your interest in contributing to Hermes Brain!

## How to Contribute

### 1. Report Bugs

If you find a bug, please create an Issue with:
- Bug description
- Steps to reproduce
- Expected behavior
- Actual behavior
- Environment information (OS, Python version, etc.)

### 2. Submit Feature Suggestions

If you have a feature suggestion, please create an Issue with:
- Feature description
- Use case
- Implementation ideas (optional)

### 3. Submit Code

1. Fork the repository
2. Create your branch (`git checkout -b feature/your-feature`)
3. Commit your changes (`git commit -m 'Add some feature'`)
4. Push to the branch (`git push origin feature/your-feature`)
5. Create a Pull Request

### 4. Code Standards

- Python code follows PEP 8 standards
- Add necessary comments
- Update related documentation
- Ensure all tests pass

### 5. Commit Standards

Use [Conventional Commits](https://www.conventionalcommits.org/) standards:

```
<type>(<scope>): <subject>

<body>

<footer>
```

Types:
- `feat`: New feature
- `fix`: Bug fix
- `docs`: Documentation update
- `style`: Code formatting (does not affect code execution)
- `refactor`: Refactoring (neither fixing a bug nor adding a feature)
- `perf`: Performance optimization
- `test`: Tests
- `chore`: Build process or auxiliary tool changes

## Development Environment

### 1. Clone Repository

```bash
git clone https://github.com/ieyz02031-source/hermes-brain.git
cd hermes-brain
```

### 2. Install Dependencies

```bash
pip install sentence-transformers
```

### 3. Run Tests

```bash
python -m py_compile scripts/*.py
```
